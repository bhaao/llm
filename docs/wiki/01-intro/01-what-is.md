# 什么是本项目

> **阅读时间**: 5 分钟  
> **适用对象**: 新用户、技术决策者

---

## 1. 项目定位

**分布式 KV 缓存系统**是一个高性能的分布式 KV 缓存系统，专为大模型推理场景设计，带哈希审计日志功能。

### 1.1 核心问题

大模型推理场景面临以下挑战：

| 挑战 | 说明 |
|------|------|
| **上下文窗口限制** | LLM 的上下文窗口有限，无法处理超长对话 |
| **KV Cache 复用** | 重复对话的 KV Cache 无法有效复用 |
| **数据完整性** | 分布式环境下 KV 数据可能被篡改 |
| **节点可信度** | 多节点环境中如何识别可信节点 |

### 1.2 解决方案

本项目采用**三层架构** + **双链设计**解决上述问题：

```
┌─────────────────────────────────────────────────────────────┐
│                    推理提供商层 (Provider Layer)             │
│  • 从记忆层读取 KV/上下文  • 执行 LLM 推理  • 写入新 KV       │
└─────────────────────────────────────────────────────────────┘
                              ↑ ↓ 读取/写入 KV
┌─────────────────────────────────────────────────────────────┐
│                    记忆层 (Memory Layer)                     │
│  • KV Cache 存储（分片、分层、压缩）  • 哈希链式校验          │
└─────────────────────────────────────────────────────────────┘
                              ↑ ↓ 哈希存证
┌─────────────────────────────────────────────────────────────┐
│                    审计日志层 (Audit Layer)                  │
│  • KV 哈希存证（不可篡改）  • 节点信誉管理  • 共识结果记录    │
└─────────────────────────────────────────────────────────────┘
```

## 2. 核心理念

### 2.1 数据本地存储 + 哈希全网存证

> **记忆层**存储实际 KV 数据，支持本地高速访问  
> **审计日志**记录 KV 哈希，提供全网存证验证

这种设计带来以下优势：

| 优势 | 说明 |
|------|------|
| **高性能** | KV 数据本地存储，无需全网广播 |
| **可验证** | 哈希全网存证，可随时验证数据完整性 |
| **防篡改** | 区块链存证不可篡改，提供可信审计 |
| **可扩展** | 数据分片存储，支持水平扩展 |

### 2.2 双链设计

| 链类型 | 存储内容 | 特点 |
|--------|----------|------|
| **区块链**（主链） | KV 哈希、元数据、信誉记录 | 不可篡改、全网共识 |
| **记忆链**（数据链） | 实际 KV 数据、版本控制 | 多副本容灾、本地缓存 |

## 3. 核心功能

### 3.1 分布式 KV 存储

- **分片**: 支持 100K+ tokens 跨节点分片
- **压缩**: zstd 压缩，93% 空间节省
- **多级缓存**: L1 CPU + L2 Disk + L3 Remote(Redis)
- **智能预取**: N-gram 模式预取

### 3.2 哈希审计日志

- **KV 哈希存证**: 每次 KV 写入自动计算哈希并上链
- **不可篡改**: 区块链存证，提供数据完整性验证
- **验证流程**: 读取 KV 数据 → 计算哈希 → 与链上存证比对

### 3.3 节点信誉系统

- **信誉评分**: 基于节点历史行为计算信誉分
- **可信调度**: 优先选择高信誉节点执行任务
- **动态调整**: 信誉分随节点行为动态调整

### 3.4 真实 LLM 集成

- **vLLM/SGLang**: 支持真实 LLM HTTP API
- **断路器模式**: 连续失败自动切换
- **指数退避**: 智能重试机制
