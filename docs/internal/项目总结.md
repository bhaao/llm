### 项目定位

> **这是一个架构验证原型，不是生产就绪系统。**

项目核心价值在于**架构设计**而非代码实现本身，展示了"区块链 + 分布式 LLM 集成"的创新思路：

- **数据本地存储 + 哈希全网存证**的双链架构
- **三层解耦**：推理提供商层、记忆层、审计日志层
- **KV Cache 优化**：分片、压缩、多级缓存、智能预取

## 架构设计

### 三层架构

```
┌─────────────────────────────────────────────────────────────┐
│                    推理提供商层 (Provider Layer)             │
│  • 从记忆层读取 KV/上下文                                    │
│  • 执行 LLM 推理计算（vLLM/SGLang API）                      │
│  • 向记忆层写入新生成的 KV                                   │
│  • 向审计日志层上报推理指标                                  │
└─────────────────────────────────────────────────────────────┘
                              ↑ ↓ 读取/写入 KV
┌─────────────────────────────────────────────────────────────┐
│                    记忆层 (Memory Layer)                     │
│  • KV Cache 存储（分片、分层、压缩）                         │
│  • 哈希链式校验（防篡改）                                    │
│  • 分布式多副本存储（容灾）                                  │
│  • 版本控制/访问授权                                         │
└─────────────────────────────────────────────────────────────┘
                              ↑ ↓ 哈希存证
┌─────────────────────────────────────────────────────────────┐
│                    审计日志层 (Audit Layer)                  │
│  • KV 哈希存证（不可篡改）                                   │
│  • 节点信誉管理                                              │
│  • 共识结果记录                                              │
└─────────────────────────────────────────────────────────────┘
```

### 双链架构

| 链类型     | 定位        | 存储内容                  | 特点         |
| ------- | --------- | --------------------- | ---------- |
| **区块链** | 全局可信存证链   | 元数据、KV 哈希存证、信誉记录、共识结果 | 不可篡改、异步提交  |
| **记忆链** | 分布式 KV 存储 | 实际 KV 数据、哈希链式串联、版本控制  | 多副本容灾、本地缓存 |

### 模块依赖关系

```
services/ (应用层，生产就绪)
    ↓
deprecated/ (遗留层，v1.0.0 删除)
    ↓
blockchain.rs (库层，单节点生产就绪)
    ↓
memory_layer.rs (库层，生产就绪)
    ↓
node_layer.rs (库层，生产就绪)
```

### 服务层架构（P11 锐评修复）

原 `ArchitectureCoordinator`（1843 行上帝对象）拆分为三个单一职责服务：

| 服务                      | 职责   | 依赖                         |
| ----------------------- | ---- | -------------------------- |
| `InferenceOrchestrator` | 推理编排 | Node/Memory/Provider Layer |
| `CommitmentService`     | 存证上链 | Blockchain                 |
| `FailoverService`       | 故障切换 | ProviderLayer + 断路器        |

## 核心技术特性

### 1. KV Cache 优化（Phase 1-2 完成）

| 优化维度  | 优化前         | 优化后                          | 提升         |
| ----- | ----------- | ---------------------------- | ---------- |
| 存储粒度  | Block-level | **Chunk-level (256 tokens)** | ✅ 细粒度      |
| 异步 IO | 同步          | **全异步**                      | ✅ 非阻塞      |
| 多级缓存  | 仅内存         | **CPU + Disk + Remote**      | ✅ 分层       |
| 预取机制  | 无           | **智能预取**                     | ✅ 实现       |
| 压缩编码  | 无           | **zstd 压缩**                  | ✅ 93% 空间节省 |
| 索引优化  | 无           | **Bloom Filter**             | ✅ O(1) 查找  |

**核心模块**:

- `kv_chunk.rs` - Chunk-level 存储（10 测试）
- `kv_index.rs` - Bloom Filter 索引（8 测试）
- `async_storage.rs` - 异步存储后端（5 测试）
- `kv_compressor.rs` - zstd 压缩（10 测试）
- `prefetcher.rs` - 智能预取（9 测试）
- `multi_level_cache.rs` - 多级缓存（7 测试）

### 2. 李群/李代数验证模块

**性能基准**（100 节点场景）:

| 指标   | 生产要求    | 实测           | 评价          |
| ---- | ------- | ------------ | ----------- |
| 聚合时间 | < 100ms | **53.19 µs** | ✅ 快 1880 倍  |
| 距离计算 | < 10ms  | **137 ns**   | ✅ 快 73000 倍 |
| 篡改检测 | ×5.47   | **∞**        | ✅ 验证通过      |

**核心模块**:

- `lie_algebra/` - 李群数学基础
- `verification_sdk.rs` - 验证 SDK
- `verifiable_package.rs` - 可验证包

### 3. 共识与同步协议

| 协议         | 状态      | 实现程度                       |
| ---------- | ------- | -------------------------- |
| **PBFT**   | ⚠️ 原型   | 三阶段提交 + 视图切换框架             |
| **Gossip** | ⚠️ 原型   | Vector Clock + Merkle Tree |
| **libp2p** | ⚠️ stub | 配置 + mDNS + GossipSub 存根   |

### 4. 故障恢复机制

**断路器模式**:

- 连续失败自动切换
- 指数退避重试策略
- 健康监控和自动恢复

**测试结果**:

- 100 线程并发测试通过
- 6 种混沌测试场景验证

### 5. 线程安全与异步 IO

**P11 锐评修复**:

- ✅ 移除 `Blockchain::Clone` 实现
- ✅ 统一使用 `Arc<RwLock<T>>` 模式
- ✅ 带超时的锁获取机制
- ✅ 锁顺序检查（避免死锁）

**异步能力**:

- ✅ `InferenceProvider` trait 异步化
- ✅ HTTP 调用原生 async/await
- ✅ 异步提交服务（Channel + 批处理 + 背压）
